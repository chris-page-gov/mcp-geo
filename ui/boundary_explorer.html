1. Welcome to Map Lab

Map Lab is a novice-first workspace for learning how UK geospatial mapping works on the web. You can search geographies, inspect UPRNs/buildings/links, test map styling, and export selector-based collections to resolved UPRN outputs.

A practical first goal (about 15 minutes): find one area, add layers, apply a hierarchy preset, create a collection with OA/LSOA/MSOA selectors, and export a CSV.

Mode Guide (important)

  • Server-proxy mode (recommended): browser requests stay on local /maps/... routes and credentials are injected server-side.
  • Direct API mode (advanced): browser calls external APIs directly and handles credentials client-side with stricter CORS/exposure constraints.
  • Verify mode in browser DevTools network: local host requests imply proxy mode.

Start Here

  • Open the Map tab and search for a boundary (for example, a ward or MSOA).
  • Refresh inventory to pull UPRNs/buildings/links for your current scope.
  • Open Collections and define reusable selector specs.
  • Export resolved UPRNs when you are ready for analysis or QA.
2. How Web Maps Work

A web map combines three building blocks:

  • Basemap style: visual context (roads, labels, land, water).
  • Data sources: your points/lines/polygons.
  • Render layers: rules that turn source data into visible symbology.

Good cartography changes emphasis by zoom level. At high zoom you can show detailed points and building geometry; at low zoom you should generalize toward line and area clarity.

Core Concepts

  • Raster vs vector tiles: vector enables dynamic styling and interaction.
  • Tile coordinates: most web maps use slippy map z/x/y tiling.
  • CRS: common UK/web references include EPSG:27700 (BNG), EPSG:3857 (web map), EPSG:4326 (lat/lon).

3. UK Geography Primer

UK geographies are purpose-driven and do not always nest perfectly. Treat statistical and administrative boundaries as complementary layers, not a single perfect hierarchy.

Common Geography Families

  • Statistical: OA, LSOA, MSOA (useful for census/indicator analysis).
  • Administrative: wards, parishes, districts, local authorities.
  • Electoral: parliamentary constituencies and related boundaries.
  • Other: built-up areas and thematic boundaries.

4. Identifier Primer

Use identifiers for selection, joins, and persistence. Names are useful for search and UI, but codes are what make workflows reproducible.

  • UPRN: unique property/address reference.
  • USRN: unique street reference.
  • GSS code: statistical/administrative geography code.
  • TOID/OSID: additional feature identifiers used in OS datasets.
Practical rule: store selections by identifier set (for example OA/LSOA/MSOA codes + explicit UPRN overrides), then resolve current membership at export time.

5. Primary Hierarchy Display Strategy

The map uses cartographic generalization: you do not lose data as you zoom out, you change what is emphasized to preserve legibility.

  • High zoom: UPRN points and building polygons for precise local detail.
  • Mid zoom: building emphasis with selective road/path links.
  • Low zoom: link and area structure for orientation and comparison.
Using hierarchy presets: select Auto by zoom to let Map Lab transition styles automatically, or use Detail-first, Balanced, and Links-first to force a learning mode.

6. Postcode Parallel Stream

Postcode workflows run in parallel to the primary hierarchy. Instead of drawing a coarse postcode outline, Map Lab highlights postcode member UPRNs through the same hierarchy logic.

  • At high zoom you can inspect member UPRNs directly.
  • At broader zooms you can emphasize buildings/links representing those members.
  • Exports can include postal-delivery filtering when delivery flags are available.
Data caveat: if delivery flags are unavailable in the indexed source, postal-delivery filtering falls back to best effort and reports a warning.

7. Styling and Cartography Lab

Styling controls are a lab: try combinations deliberately and evaluate clarity, not just visual effect. Good thematic maps preserve context while making selected features obvious.

Recommended Experiments

  • Switch basemap style and compare boundary readability.
  • Toggle UPRN density versus UPRN dots and inspect overplotting.
  • Test hierarchy presets at multiple zoom bands.
  • Enable buildings then roads/paths to compare urban/rural legibility.
Recipe: improve clarity quickly
  • Use a lighter basemap style.
  • Keep one dominant highlight color for selected geography.
  • Avoid maximum opacity on all layers at once.

8. Build Your Own Collections

Collections are selector specifications, not materialized UPRN snapshots. This keeps them compact, editable, and up to date as source memberships evolve over time.

Collection Model

  • Selectors: gss_code, postcode, uprn, polygon.
  • UPRN overrides: explicit include and exclude lists.
  • Export: resolve selectors, apply overrides, emit canonical and membership columns.

Workflow

  • Create collection and set name/description.
  • Add OA/LSOA/MSOA/LAD selectors using a GSS level picker.
  • Add optional explicit UPRNs for corrections or edge cases.
  • Export resolved UPRNs and poll async job status until complete.
Export default schema: uprn, postcode, oa_code, local_authority_name, plus canonical geography and selector-membership columns when available.
9. Troubleshooting

Quick Checks

  • Confirm you are in server-proxy mode for simplest setup.
  • Check request host in DevTools network panel.
  • Use Refresh inventory after changing scope or boundary selections.

Common Symptoms

  • 429 Too Many Requests: local rate limit reached; reduce burst traffic or adjust server limits/exemptions.
  • 401/403: credential missing/invalid for OS-backed endpoints.
  • No features returned: selector mismatch, empty area, or filtered scope.
  • CRS mismatch look/offset: verify expected projection and source assumptions.

10. Glossary
  • Basemap: contextual reference map beneath thematic overlays.
  • GSS code: official code for a UK statistical/administrative geography.
  • Hierarchy preset: zoom-aware styling mode for feature emphasis.
  • LAD: local authority district code/area.
  • MSOA / LSOA / OA: statistical output-area geography levels.
  • Postcode stream: postcode member highlighting through UPRN membership.
  • UPRN: Unique Property Reference Number for an address/property.
  • USRN: Unique Street Reference Number for a street entity.
  • Vector tile: tiled vector geometry that can be re-styled client-side.
