# Release 0.2.3 - 2026-01-21

Focus: Claude stdio compatibility, traceability, and containerized usage.

## Highlights
- Added MCP stdio trace proxy and UI interaction logging for end-to-end capture.
- Added Dockerfile + `.dockerignore` for containerized STDIO deployment.
- Improved stdio adapter interoperability (framing, initialize response, tool name normalization).
- Fixed stdio tool call payload handling (`arguments` support).

## Added
- JSON-RPC trace proxy (`scripts/mcp_stdio_trace_proxy.py`).
- UI event logging tool (`os_apps.log_event`) with `UI_EVENT_LOG_PATH`.
- Client tracing guide (`docs/client_trace_strategy.md`).
- Docker build for stdio usage plus Claude config docs.

## Changed
- Geography selector MCP-App now emits UI interaction events for tracing.

## Fixed
- Auto-detect JSON line framing vs Content-Length for stdio.
- Initialize response includes protocol version + serverInfo (spec-style).
- Tool names normalized for Claude while still accepting dotted names.
- `ons_data.create_filter` no longer marked read-only.
- `tools/call` accepts `arguments` payloads.

## Upgrade Notes
- Rebuild the Docker image after upgrading (`docker build -t mcp-geo-server .`).
- If your client expects JSON lines, set `MCP_STDIO_FRAMING=line` (auto-detect usually works).

## Checksums & Integrity
No binary artifacts. Tag: `v0.2.3`.
