# Release 0.2.2 - 2026-01-20

Focus: tool search + MCP-Apps UX, evaluation harness, and OS Places bbox/WGS84 improvements.

## Highlights
- Added tool search (`/tools/search` and stdio `tools/search`) with annotations and defer-loading hints.
- Introduced `os_mcp.route_query` and `os_mcp.descriptor` to guide tool selection.
- MCP-Apps UI resources upgraded (MapLibre geography selector) and tutorial expanded for multi-client setup.
- Evaluation harness + rubric landed under `tests/evaluation/` with audit logs and docs.
- OS Places now handles WGS84 coords correctly and auto-tiles/clamps oversized bbox queries.
- Health check endpoint renamed to `/health` (breaking change from `/healthz`).

## Added
- Evaluation framework, docs, and audit logging for benchmarks.
- Skills resource (`skills://mcp-geo/getting-started`).
- MCP-Apps UI resources and helper tools (`os_apps.render_*`).
- Tool catalog generator and published `docs/tool_catalog.md`.
- CSV/XLSX output for `ons_data.get_filter_output`.

## Changed
- STDIO adapter relocated to `server/stdio_adapter.py` with legacy wrapper retained.
- `mcp.json` server entries renamed to `mcp-geo-stdio` / `mcp-geo-http`.
- OS Places tools request WGS84 output by default; `os_places.within` now tiles/clamps large bbox.

## Fixed
- WGS84 axis order for OS Places `nearest`/`within` to avoid BNG coverage errors.
- Non-200 OS API responses normalized to consistent 501 envelopes.
- JSON-RPC invalid params handling for non-dict `params`.

## Upgrade Notes
- Update clients to use `/health` (or add a compatibility alias if needed).
- If you rely on `mcp.json`, update server IDs to `mcp-geo-stdio` / `mcp-geo-http`.
- Oversized bbox calls to `os_places.within` will now tile or clamp for safety.

## Checksums & Integrity
No binary artifacts. Tag: `v0.2.2`.
