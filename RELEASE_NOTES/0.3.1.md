# Release Notes 0.3.1 (2026-02-13)

## Added
- Added a curated `starter` discovery toolset and default toolset filter knobs:
  - `MCP_TOOLS_DEFAULT_TOOLSET`
  - `MCP_TOOLS_DEFAULT_INCLUDE_TOOLSETS`
  - `MCP_TOOLS_DEFAULT_EXCLUDE_TOOLSETS`
- Added `os_mcp.select_toolsets` for post-initialization discovery scoping with
  optional query-driven toolset inference.
- Added a VS Code toolset template in-repo at
  `.vscode/mcp-geo.toolsets.jsonc` and documented how to install it into VS
  Code user prompts.
- Added troubleshooting guidance for OS VTS custom symbol-layer label
  limitations and documented HTML/DOM markers as the preferred overlay
  approach.

## Changed
- Updated HTTP and MCP discovery surfaces to apply default toolset filters when
  clients do not pass explicit `toolset` / `includeToolsets` /
  `excludeToolsets` parameters.
- Extended initialize-time STDIO and Streamable HTTP logging with compact client
  capability/protocol support summaries for traceability.
- Added MCP elicitation support for `os_mcp.select_toolsets` in both STDIO and
  Streamable HTTP transports.
- Tightened large-payload handling for Claude/STDIO map workflows:
  - explicit limits on `os_places.search` and `os_names.find`
  - lower default/max limits for `os_map.inventory`
  - truncated STDIO text `content` with guidance to use structured `result.data`

## Validation
- No additional test run executed for this documentation/version bump release.
  Latest verified suite remains recorded in `CONTEXT.md`.
