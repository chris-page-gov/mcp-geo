# Release 0.2.1 - 2025-09-17

Focus: MCP STDIO adapter resource parity (ETag + describe) and developer client ergonomics.

## Highlights
- STDIO `resources/read` now supports weak ETag generation and conditional retrieval via `ifNoneMatch` producing `{ "notModified": true }` minimal responses when unchanged.
- New STDIO `resources/describe` method returns resource metadata (name, description, license).
- Developer client (`scripts/mcp_client.py`) adds REPL mode, generic JSON param parsing, initial log notification suppression, and `--if-none-match` convenience flag.
- ETag computation centralized in adapter for consistency across variants.
- Test suite expanded to cover ETag round trips, describe method, client conditional flows (adds scripts & tests ensuring framing correctness).

## Added
- STDIO adapter: `resources/read` parity enhancements with ETag + conditional support.
- STDIO adapter: `resources/describe` method.
- Client: REPL mode, generic params, log notification skip, `--if-none-match` flag.

## Changed
- Resource STDIO responses now always include an `etag` when returning full payload; conditional matches return minimal structure.
- Centralized ETag helper for maintainability and correctness.

## Fixed
- STDIO tests updated to tolerate initial log frames preventing early KeyErrors.

## Upgrade Notes
No breaking schema changes. To leverage conditional resource retrieval, pass prior `etag` as `ifNoneMatch` param or via client flag.

## Checksums & Integrity
No binary artifacts. Tag: `v0.2.1`.
