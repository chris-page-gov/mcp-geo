# Release Notes 0.2.11 (2026-02-06)

## Added
- Admin lookup level filtering, match modes, and live fallback for cache search.
- NOMIS query error detection for non-JSON and upstream error payloads.
- Stats routing guidance for comparisons and small-area caveats.
- STDIO schema normalization for sanitized tool names and UI fallbacks for the stats dashboard.

## Changed
- Admin lookup search ordering now prioritizes cleaner town-level matches.
- Tool search prompt guidance now favors `os_mcp.route_query` and level-filtered admin lookup.
- MCP-Apps render tools continue to expose `resourceUri`, `uiResourceUris`, and `_meta.ui.resourceUri`; `resource_link` content blocks are now opt-in via `MCP_APPS_RESOURCE_LINK` to avoid unsupported format warnings in Claude.
- MCP client capabilities are logged during initialize in both stdio and HTTP paths for UI diagnostics.

## Tests
- Added coverage for admin lookup level filtering and NOMIS query error handling.
- Expanded evaluation harness coverage for NOMIS tooling and stats routing.
- Added coverage for NOMIS client error handling, admin cache fallback, and stdio UI fallbacks.
