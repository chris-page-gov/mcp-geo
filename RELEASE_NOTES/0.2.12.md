# Release Notes 0.2.12 (2026-02-11)

## Added
- `scripts/rate_limit_assessor.py` to calibrate `RATE_LIMIT_PER_MIN` from observed 429 ratio and latency.
- Endpoint matrix and upstream endpoint-contract regression tests across HTTP endpoints and tool families.
- Protocol negotiation regression suite (`tests/test_protocol_versions.py`) and extended stdio/http negotiation coverage.
- Playground version visibility matrix (including MCP Apps protocol visibility) and smoke-test coverage for it.

## Changed
- Raised default `RATE_LIMIT_PER_MIN` from 120 to 207 based on local calibration runs.
- Added per-task 429 summaries in evaluation audit output to track backoff reliance (`429 Rate-limit hits` and by-tool breakdown).
- Upgraded MCP core protocol default to `2025-11-25` and implemented explicit version negotiation for compatible clients (`2025-11-25`, `2025-06-18`, `2025-03-26`, `2024-11-05`).
- HTTP transport now validates `MCP-Protocol-Version` against supported/negotiated session versions and returns `mcp-protocol-version` response headers.
- `os_mcp.descriptor` now advertises `supportedProtocolVersions` and `mcpAppsProtocolVersion` (`2026-01-26`).

## Fixed
- Normalized invalid upstream JSON handling across OS, ONS, and admin lookup clients to `502` with `UPSTREAM_INVALID_RESPONSE`.
- `nomis.query` geography resolution now handles common Census GSS geography codes via NOMIS geography lookups.
- Playground Playwright config now uses a dedicated strict port to reduce local flaky runs due to port collisions.

## Validation
- Local test baseline previously validated in this release stream: `pytest -q` with coverage gate passing.
- Playground checks pass after version-visibility updates:
  - `npm run build`
  - `npm test -- --workers=1`
