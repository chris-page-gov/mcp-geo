# Release 0.3.2

Date: 2026-02-17

## Summary

`0.3.2` publishes the completed map-delivery implementation program and
stabilizes MCP client interoperability for mixed host environments.

## Highlights

- Completed map-delivery workstreams (`MDR-I*`, `MDR-N*`, `MDR-M*`, `MDR-E*`)
  with synchronized docs, tooling, resources, and tests.
- Added offline map delivery capabilities with explicit handoff contracts:
  - `os_offline.descriptor`
  - `os_offline.get`
  - `resource://mcp-geo/offline-packs-index`
  - `resource://mcp-geo/offline-packs/*`
- Added secure binary retrieval for offline packs:
  - `GET /resources/download?uri=resource://mcp-geo/offline-packs/<file>`
- Expanded map-delivery evidence artifacts:
  - host simulation matrix
  - map quality checks + waivers
  - story gallery report and slide content
  - scenario-pack export from notebook sources
- Hardened interoperability behavior:
  - consistent `structuredContent` response shaping
  - expanded tool-name alias handling (display-style + namespaced forms)
  - safer offline payload handling (inline threshold + external download URL)
  - artifact-stable offline hash behavior with digest caching fallback

## Validation

- `pytest -q`
  - 760 passed
  - 6 skipped
  - 90.14% coverage

## Notes

- `resources/offline_map_catalog.json` now includes `sha256` values for bundled
  offline packs.
- Devcontainer `postCreateCommand` now uses `python3 -m pip` for interpreter
  consistency.
