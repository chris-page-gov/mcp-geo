# Release 0.7.0

Date: 2026-03-16

## Summary

`0.7.0` packages the post-`0.6.0` platform hardening work across routing,
security, MCP resource delivery, stakeholder evaluation, and reporting.

This release turns several previously partial or host-fragile workflows into
first-class, test-backed surfaces: route planning on the MRN graph, authenticated
resource recovery across MCP transports, hardened playground workbenches,
dataset-aware NOMIS geography recovery, and a governed DOCX hygiene baseline for
public-facing authored documents.

## Highlights

- Added the OS MRN route-planning stack with `os_route.descriptor`,
  `os_route.get`, pgRouting/PostGIS graph services, route-graph bootstrap
  assets, and updated stakeholder benchmark routing coverage.
- Added authenticated cross-transport MCP resource fallback via
  `os_resources.get`, normalized `resourceHandoff` metadata, transport-aware
  `resource_link` responses, and MCP HTTP auth parity for raw resource/tool
  routes.
- Added the OWASP MCP validation pack, signed tool-manifest workflow,
  deployment/monitoring attestations, and strict baseline evidence with a
  compliant `100.0` score.
- Added the hardened multi-workbench playground shell with deterministic full
  Playwright coverage, live smoke coverage, benchmark resources, DSAP workbench
  support, and review-driven bridge hardening.
- Added dataset-aware NOMIS geography recovery and current-source admin lookup
  refreshes, including Harold Wood ward recovery to current code `E05013973`.
- Added the DOCX hygiene policy, sanitizer/checker automation, first-pass
  metadata cleanup of repo-authored public DOCX files, and published hygiene
  audit outputs.
- Added the analytical-index publication set, refreshed stakeholder benchmark
  pack/live rerun evidence, and the Road Stat USRN integration task brief.
