# Release 0.6.0

Date: 2026-03-08

## Summary

`0.6.0` packages the validated Map Lab merge set and the latest reporting and
benchmarking work on top of the public `0.5.0` release.

This release expands the browser-facing mapping workflow, improves Docker-backed
host launchers for Codex and Claude, and publishes new geospatial report outputs
and benchmark artifacts.

## Highlights

- Added shared Docker-backed stdio launcher `scripts/mcp-docker-local` plus the
  Codex-specific wrapper `scripts/codex-mcp-local`, while retaining
  `scripts/claude-mcp-local` for Claude-first defaults.
- Added the Codex-vs-Claude host benchmark harness, scenario pack, runbook, and
  host-aware trace/report outputs.
- Added Teignmouth, Exmouth, and Sidmouth wheelchair-access map/report
  artefacts plus reusable export-driven generation in
  `scripts/generate_teignmouth_wheelchair_access_map.py`.
- Added the public MCP-Geo functionality showcase generation pipeline and
  published markdown, DOCX, and PDF outputs.
- Added the full Map Lab selector workflow on the compatibility-locked boundary
  explorer surface, including Help/Map/Collections tabs, async selector export
  polling via `os_map.get_export`, ONS UPRN reverse-lookup indexing, guidance
  and cache diagnostics, and expanded Playwright boundary coverage.
- Hardened storage/cache/bootstrap behavior for Docker-backed and devcontainer
  workflows, including named-volume PostGIS support and cold-start cache guards.
